# ADR-001 - Architecture en couches

## Statut

Acceptée

## Contexte

AllenVoice est un projet de synthèse ayant vocation à devenir un produit SaaS.

Le système devra évoluer facilement avec l'ajout de nouvelles fonctionnalités (nouveaux fournisseurs IA, nouvelles fonctionnalités du tableau de bord, etc.).

## Décision

Le backend sera organisé selon une architecture en couches :

- Controllers
- Services
- Repositories
- Database

## Justification

Cette architecture permet de séparer les responsabilités, de faciliter les tests, la maintenance et l'évolution du logiciel.

## Conséquences

### Avantages

- Code plus lisible
- Maintenance simplifiée
- Facilité d'évolution

### Inconvénients

- Plus de fichiers
- Légère complexité supplémentaire
